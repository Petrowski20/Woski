# Woski Documentation

This directory contains the technical and product documentation for Woski.

The documentation is organized by topic rather than by implementation to make it easier to understand how the project evolves over time.

---

# Documentation Structure

## 00 - Overview

High-level information about the project.

Contents:

- Product vision
- Long-term roadmap
- Project glossary

---

## 01 - Architecture

Architecture and domain design.

Contents:

- Overall architecture
- Domain model
- System organization

---

## 02 - Database

Database documentation.

- Target schema

---

## 03 - Frontend

Frontend architecture and conventions.

Contents:

- Routing
- UI organization
- Components

---

## 04 - Backend

Backend services and infrastructure.

Contents:

- Supabase
- Future API
- Authentication
- Background workers

---

## 05 - Architecture Decision Records

Historical record of the most important technical and product decisions.

Each ADR documents:

- Context
- Decision
- Consequences
- Alternatives considered

---

## Backlog

The project backlog is maintained in `backlog.md`.

Features should be organized as:

- Epic
- Feature
- Task

---

# Documentation Principles

- Documentation evolves together with the project.
- Important decisions are recorded as ADRs.
- Architecture is documented before implementation whenever possible.
- Documentation should describe the "why", not only the "how".
